# Задание 1

В файле variables.tf определены входные переменные, необходимые для работы с провайдером Yandex Cloud и настройки создаваемых ресурсов. Они делают конфигурацию Terraform гибкой и переиспользуемой – значения можно задать через файл terraform.tfvars, переменные окружения или напрямую при запуске.

В исходном проекте (после исправления синтаксиса) были объявлены следующие переменные:

cloud_id – идентификатор облака, в котором будут создаваться ресурсы.

folder_id – идентификатор каталога внутри облака.

default_zone – зона доступности (по умолчанию ru-central1-a). Используется провайдером и ресурсами, требующими привязки к зоне.

default_cidr – список CIDR-блоков для подсети (по умолчанию ["10.0.1.0/24"]).

vpc_name – имя для сети и подсети (по умолчанию "develop").

vms_ssh_root_key – публичный SSH-ключ, который будет вставлен в метаданные виртуальной машины для доступа пользователю ubuntu.

Эти переменные напрямую подставляются в блоки провайдера и ресурсов:

В provider "yandex" используются cloud_id, folder_id, default_zone и путь к сервисному ключу (жёстко заданный как ~/.authorized_key.json).

В yandex_vpc_network и yandex_vpc_subnet используются vpc_name и default_cidr.

В yandex_compute_instance – default_zone, vms_ssh_root_key (для metadata.ssh-keys) и core_fraction, preemptible (зафиксированы в коде).

Таким образом, изучив variables.tf, можно легко поменять регион, сетевые настройки или ключ доступа, не редактируя основную логику в main.tf.
